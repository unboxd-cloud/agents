- Manual support
- Complex integrations

## Defensibility Model

```text
Defensibility
=
Open-source adoption
+ operational trust
+ reusable delivery system
+ governance depth
+ ecosystem distribution
```

### Open-source adoption

Creates visibility, credibility, and technical trust through repositories, templates, reference architectures, deployment guides, and community usage.

### Operational trust

Customers stay because Unboxd Cloud runs systems reliably through monitoring, backups, incident response, security updates, and support.

### Reusable delivery system

Margins improve because each customer is not built from scratch. Reusable assets include blueprints, sector packages, automation, GitOps templates, and standard runbooks.

### Governance depth

Enterprise value increases when identity, policy, audit, agent governance, risk controls, and compliance evidence become embedded.

### Ecosystem distribution

Partners and developers expand reach through marketplace listings, certified templates, solution catalogs, and partner-led delivery.

Clean statement:

> Unboxd Cloud becomes defensible by combining open-source adoption, reusable platform delivery, managed operational trust, enterprise governance depth, and partner ecosystem distribution.
