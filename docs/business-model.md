## Maturity Model

### CNCF-Mature

Prefer CNCF graduated and incubating projects for the production foundation. Use sandbox or non-CNCF tools only when they fill a clear gap and are governed.

### Cloud-Native Mature

Designed for scalable, resilient, observable, secure, automated, and portable operations across cloud, edge, and multi-cloud environments.

Includes:

- Containers
- Service discovery
- Ingress
- Certificates
- GitOps
- Observability
- Security
- Secrets
- Policy
- Cost visibility
- Backup and restore
- Resilience
- Scaling
- Automation

### Kubernetes-Mature

Not just running on Kubernetes — designed, operated, secured, observed, upgraded, and governed the Kubernetes way.

Checklist:

- Containerized workloads
- Helm / Kustomize / manifests
- Health checks
- Resource requests and limits
- Ingress
- TLS
- Secrets externalized
- Persistent storage
- Backup and restore
- Metrics
- Logs
- Traces
- RBAC
- Network policies where needed
- GitOps compatibility
- Upgrade path
- Rollback path
- Runbooks

### Enterprise-Ready

Secure, governed, auditable, observable, supportable, upgradeable, recoverable, cost-visible, policy-controlled, and identity-aware.

Checklist:

- SSO / SCIM
- RBAC / ABAC / ReBAC
- Audit logs
- Policy-as-code
- Secrets management
- Backup and restore
- SLA / SLO
- Monitoring and alerting
- Cost visibility
- Incident response
- Access reviews
- Compliance evidence
- Data residency options
- Private deployment
- Support tiers
- Runbooks
- Upgrade path
- Security review
